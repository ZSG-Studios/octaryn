Octaryn Texture Packs

Texture packs become module assets.

The current texture-pack path imports Minecraft-style resource pack ZIPs into basegame-owned atlas files. The API sees declared assets; the client sees generated textures; the server never owns presentation data.

Pipeline

octaryn-basegame/Tools/build_atlas_from_pack.py is the current import path. It reads a resource pack ZIP, extracts known block textures, applies current basegame tint rules, and writes aligned atlas outputs.

python3 octaryn-basegame/Tools/build_atlas_from_pack.py \
  --pack /path/to/resource-pack.zip \
  --sha256 PACK_SHA256 \
  --pack-name "PACK_NAME" \
  --pack-source-url PACK_SOURCE_URL \
  --pack-license-name "PACK_LICENSE" \
  --pack-license-url PACK_LICENSE_URL \
  --pack-license-file "path/to/included/license.txt" \
  --pack-credit "PACK_AUTHOR_OR_TEAM" \
  --pack-credits-url PACK_CREDITS_URL \
  --cache-dir build/dependencies/texture-packs \
  --output build/debug-linux/basegame/assets/atlases/basegame-color.png \
  --normal-output build/debug-linux/basegame/assets/atlases/basegame-normal.png \
  --specular-output build/debug-linux/basegame/assets/atlases/basegame-specular.png \
  --animation-output build/debug-linux/basegame/assets/atlases/basegame-animation.png \
  --animation-manifest build/debug-linux/basegame/assets/atlases/basegame-animation.txt \
  --tile-size 32 \
  --layer-count 29

Use an explicit pack path, SHA-256, and license metadata when preparing release assets. If --pack is omitted, the tool uses its configured default download URL and records the default pack license metadata.

Default Source

The current default source is Classic Faithful 32x Jappa. It is useful for local atlas generation, but it is not public-domain content and is not covered by Octaryn's source license.

Field Value
Pack Classic Faithful 32x Jappa
License Faithful License Version 3
Credit Faithful Resource Pack and ClassicFaithful contributors.
Credits Classic Faithful credits

Pack Input

The importer expects Minecraft-style block texture paths inside the ZIP.

Input Purpose
assets/minecraft/textures/block/*.png Primary block texture path.
assets/minecraft/textures/blocks/*.png Legacy block texture path also accepted by the importer.
*_n.png Optional normal map for the matching color texture.
*_s.png Optional specular map for the matching color texture.
*.png.mcmeta Optional animation metadata for animated texture frames.

Atlas Layers

Block data references atlas layer numbers by face. Color, normal, specular, and animation outputs stay aligned by layer.

Layer Range Current Use
0 Transparent empty layer.
1-8 Grass, dirt, stone, sand, snow, and log faces.
10-15 Leaves, flowers, and bush sprites.
16, 26 Water still and water flow layers.
17-23 Torch sprite layers with color tinting.
24-25 Planks and glass.
27-28 Lava still and lava flow layers.

Generated Outputs

Output Content
--output Color atlas.
--normal-output Normal atlas. Missing normals use a flat fallback.
--specular-output Specular atlas. Missing specular maps use a neutral fallback.
--animation-output Atlas containing extracted animation frames.
--animation-manifest Layer, frame count, first frame, and frame tick metadata.
*.txt sidecar Generated manifest with pack path, source, license, credit, layer count, tile size, outputs, and warnings.

Manifest Assets

Generated atlas files are declared through module asset declarations. Current valid asset kinds include atlas and texture.

AssetDeclarations:
[
  {
    "AssetId": "octaryn.basegame.texture.atlas.color",
    "AssetKind": "atlas",
    "RelativePath": "Assets/Atlases/basegame-color.png"
  },
  {
    "AssetId": "octaryn.basegame.texture.atlas.normal",
    "AssetKind": "atlas",
    "RelativePath": "Assets/Atlases/basegame-normal.png"
  }
]

Ownership Rules

Owner Texture-Pack Responsibility
octaryn-basegame Pack import rules, atlas layer map, content data, generated basegame assets, and basegame-specific tooling.
octaryn-shared Asset declaration contracts only. No image loading, pack parsing, or renderer behavior.
octaryn-client GPU upload, shader sampling, atlas config, and presentation.
octaryn-server No texture-pack presentation work. Server authority consumes content IDs and gameplay rules, not GPU assets.

License And Credit

Texture packs are third-party content. Do not import, commit, publish, or package pack assets unless the pack license permits that use. Release packages should keep upstream license files, credit text, source URL, version or commit, and SHA-256 beside the generated asset manifest.

For Faithful-licensed work, release materials must include clear credit, link back to Faithful, avoid monetizing content containing their work, avoid implying official Faithful status, and include the unmodified Faithful license file with any distributed content that contains their textures.
