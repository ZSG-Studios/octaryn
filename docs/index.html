Octaryn Documentation

Owner-split API and architecture docs.

Octaryn is being ported from a monolithic engine tree into a clean AAA modular platform with strict separation between client presentation, server authority, shared contracts, and basegame implementation.

Open API Docs View Examples GitHub Repository

Owner Boundaries

Client

octaryn-client/ owns windowing, input, rendering, shaders, overlays, prediction, and client host integration.

Server

octaryn-server/ owns authoritative simulation, validation, persistence, ticks, replication, and transport hosting.

Shared

octaryn-shared/ owns implementation-free contracts, IDs, snapshots, manifests, capability IDs, host API IDs, and validation policy.

Basegame

octaryn-basegame/ owns bundled gameplay, content declarations, assets, data, and basegame-specific tools.

API Policy

The API boundary is deny-by-default. Modules use explicit shared contracts and approved capabilities. They do not see client rendering internals, server persistence internals, native bridges, raw threading, transport internals, or broad host services.

  • capability-scoped
  • host-owned scheduling
  • implementation-free shared contracts
  • module validation before activation
  • owner-partitioned outputs
